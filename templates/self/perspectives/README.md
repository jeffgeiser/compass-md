# Perspectives

One file per topic I have stated views on. Used by agents reasoning about decisions or producing content where my perspective should be reflected.

---

## When to create a new perspective file

Create a file when:
- I have an actual position on the topic, not just passing thoughts
- The position is one I'd defend in conversation
- An agent acting on my behalf would benefit from knowing the position

Don't create a file just because the topic is interesting. The substrate works because perspectives reflect actual views, not aspirational ones.

## File naming

Use kebab-case, descriptive names: `competitive-strategy.md`, `ai-safety.md`, `management.md`, `pricing-models.md`.

## File structure

Each perspective file should have:

```markdown
# [Topic]

## My view

[The actual position, in 1-3 sentences]

## Why I hold it

[The reasoning, with examples or citations where possible]

## Where I differ from common takes

[How my view differs from the conventional wisdom or common opinion]

## What would change my mind

[Conditions or evidence that would update this view — important for agents to know when to flag new information]

## Related perspectives

- [Cross-references to other perspective files]

## Last updated

[YYYY-MM-DD]
```

## Maintenance

Review perspectives periodically. Views evolve. When a view changes:
- Don't delete the old version
- Update the file with the new view
- Note the change in `decisions.md` if it's significant
- Note the date in "Last updated"
