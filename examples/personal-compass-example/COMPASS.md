# Compass Schema

This Compass follows the [compass-md convention](https://github.com/jeffgeiser/compass-md) v0.1.

---

## Owner

Alex Chen — founder and CEO of Foundry (B2B SaaS, infrastructure observability)

## Active modules

- `self/`: yes  ← personal context (voice, perspectives, preferences, facts)
- `team/`: no   ← Alex uses a separate team Compass for Foundry's sales team

## Conventions

- Perspective files use the standard structure but Alex sometimes adds a "What I got wrong" section when views have shifted
- Refinements should be proposed after meaningful corrections, not after single-word edits

## Refinement workflow

- Refinements are proposed by agents to `refinements/pending/`
- Alex reviews pending refinements on Sunday mornings (weekly)
- Approved refinements applied within 24 hours of review
- Refinement frequency: maximum 3 per session; propose only high-confidence refinements unless explicitly asked
- Low-confidence proposals should be queued with an explicit ask ("Worth capturing?" at end of session)

## Engagement notes

- Be direct. If I'm wrong, say so.
- Don't pad responses. I can read; I don't need things repeated.
- Concrete over abstract. I'll ask for the framework if I want it.
- When I'm working through a decision, present options with tradeoffs, not a recommendation. I'll decide.
- Don't use "circle back," "leverage," "synergy," "touch base," or similar corporate filler.

## Sensitive content

- Never include customer names, ARR specifics, or deal details
- Never include fundraising details or investor names
- Compensation data and equity details stay out of the Compass
- Don't include anything a competitor could use

## Agent instructions

When acting on behalf of Alex:

1. **Read COMPASS.md first** to understand this Compass's conventions
2. **Identify relevant files** for the current task:
   - Drafting external content → read `self/voice.md`, relevant `self/perspectives/`
   - Making recommendations → read `self/preferences.md`, relevant `self/perspectives/`
   - Understanding context → read `self/facts.md`
   - Recalling decisions → read `self/decisions.md`
3. **Read those files** before generating output
4. **Act with that context** informing your output
5. **Cite which Compass files** informed your output
6. **If you observe something** worth capturing as a refinement, propose it to `refinements/pending/` per the refinement format
7. **Append to log.md** for significant events
